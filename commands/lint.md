# /roundtable:lint

Read-only docs sweep. Rebuilds `<docs_root>/INDEX.md`. Reports issues; does not auto-fix. Output language follows the project's CLAUDE.md convention.

## Step 1: Read context

If `$ARGUMENTS` is an absolute path or `.`, use that as `target_project`. Otherwise read `docs_root` from session start context. If `docs_root` isn't set, abort with a one-line message asking the user to invoke from inside the target project or pass a path.

## Step 2: Rebuild INDEX.md

Scan every `.md` under these 7 directories:

`<docs_root>/{analyze,design-docs,exec-plans/active,exec-plans/completed,testing,reviews,bugfixes}/`

For each file, read frontmatter `slug` (or filename) and the first H1 heading. Write `<docs_root>/INDEX.md` with one section per directory, listing entries as:

```
- [<slug>](<rel-path>) — <first-H1>
```

Header:
```
# Documentation Index

> Auto-generated by /roundtable:lint. Manual edits will be overwritten.
> Last updated: <YYYY-MM-DD>
```

## Step 3: Issue checks

**Critical** (must fix):
- Broken internal links: any `[text](path)` or `[[page]]` whose target doesn't exist
- exec-plan with `source:` pointing to a non-existent design-doc

**Warning** (should fix):
- Orphan files: a `.md` under one of the 7 dirs that no other doc links to and that doesn't appear in INDEX.md (after rebuild)
- Stale active exec-plans: not modified in >30 days (use `git log -1 --format=%cs <file>`); suggest moving to `completed/`
- Fully-checked active exec-plans: every `- [ ]` is now `- [x]`; suggest moving to `completed/`

**Info**:
- Docs without an H1 heading
- Docs without a `slug:` frontmatter

## Step 4: Output

```
# Documentation Health Report

> project: <project_id>
> date: <YYYY-MM-DD>
> docs_root: <docs_root>

## Critical
- <path:line> — <issue> → <suggested action>

## Warning
- ...

## Info
- ...

## Stats
- INDEX.md rebuilt: <N> entries
- Critical: <X>
- Warning: <Y>
- Info: <Z>
```

## Forbidden

- Modifying any file other than `<docs_root>/INDEX.md`
- Moving exec-plans (only suggest)
- Auto-fixing any reported issue
